# コミット

## 実行手順

### 1. 変更の分析
スキル呼び出し時に注入されるコンテキスト（git status・git diff）を使う。提供されていない場合のみ `git diff` を実行する。

### 2. 変更のグルーピング
- 論理的に分類: 機能追加、バグ修正、リファクタリング、スタイル修正など
- 1 コミット = 1 論理変更の原則

### 3. コミット作成
- セキュリティ: `.env`, `credentials.json` 等はステージしない（該当する場合は警告）
- グループごとに独立したコミットを作成
- Conventional Commits 形式で日本語メッセージを記述

## 注意事項

- pre-commit hook でエラーが発生した場合はコミットを中断する
- エラーをすべて修正してから再試行すること
